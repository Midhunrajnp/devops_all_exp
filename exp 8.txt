# 🧪 Experiment 8: Jenkins Build Triggers

**Tasks:** Configure SCM polling and GitHub webhook
**Outcome:** Automatic build triggering

We’ll do this **fresh like an exam**, with a simple Java repo + Jenkins Freestyle Job.

---

# 🔥 What this experiment means

Instead of clicking **Build Now** manually:

* Jenkins automatically checks GitHub changes
* Jenkins automatically starts build

Two methods:

1. **SCM Polling** → Jenkins checks GitHub repeatedly
2. **GitHub Webhook** → GitHub instantly notifies Jenkins

# 🎯 Viva Answers

### ❓ What is SCM Polling?

> Jenkins periodically checks repository changes.

---

### ❓ What is Webhook?

> GitHub directly notifies Jenkins when code changes occur.

---

### ❓ Difference?

| SCM Polling    | Webhook         |
| -------------- | --------------- |
| Periodic check | Instant trigger |
| Slower         | Faster          |

---

### ❓ Which is better?

> Webhook because it is event-driven.
